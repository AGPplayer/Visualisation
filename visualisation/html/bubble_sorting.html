Bubble sorting

The algorithm iterates through the list, comparing neighboring elements. If the current element is larger than the next, they swap places. As a result of each pass, the largest item "floats" to the end of the list, like an air bubble in water.

  • Difficulty:

    In the worst and average cases. This makes it slow on large datasets.

  • Plus:

    Simplicity of implementation and minimal memory costs.

  • When to use it:

    Only for educational purposes or for very small arrays where speed is not critical.
